# Build a Todo List

This guide walks through building a working todo list from scratch. Each todo item is its own entity — toggling, deleting, and rendering all operate on individual entities rather than a shared array. Along the way you'll use:

- DOM rendering (`DOMElement`, `TextContent`, `Classes`, `Style`)
- interactions (`Clickable`, `Clicked`)
- entity-per-item state with markers (`Done`)
- parent/child entity relationships
- reactive systems for lifecycle and updates

## 1. Define Your Components

Each todo item will be an entity with a `TodoText` component. Completed items get a `Done` marker. Markers tag the other key elements so systems can find them:

<div data-code-tabs>

```tsx
import { defineComponent, defineMarker } from "@ecs-test/ecs";

// TodoText holds the actual text. Done is a marker — its
// presence/absence IS the completed state. No booleans needed.
const TodoText = defineComponent<{ value: string }>("TodoText");
const Done = defineMarker("Done");

// Markers to tag key UI entities so systems can find them
const TodoInput = defineMarker("TodoInput");
const AddButton = defineMarker("AddButton");
const TodoListContainer = defineMarker("TodoListContainer");
const ToggleButton = defineMarker("ToggleButton");
const DeleteButton = defineMarker("DeleteButton");
```

</div>

Notice there's no `Todos` array or `completed: boolean` field. Each todo is its own entity — `TodoText` holds the text, and the presence or absence of `Done` is the completed state. This is the ECS way: state is expressed as components on entities, not data in objects.

## 2. Build the Static UI Shell

The app has a heading, an input row, and an empty container where todo entities will appear as children:

## 3. Render Todo Entities on Enter

When a `TodoText` component is added to an entity, a system gives it a DOM element and builds its child entities (toggle button, label, delete button):

<div data-code-tabs>

```tsx
import { defineReactiveSystem, Entities, materialize } from "@ecs-test/ecs";

// Build the DOM children for a single todo row.
// Called on initial render and when Done state changes.
function buildTodoRow(world, todoEntity) {
  const text = world.get(todoEntity, TodoText)?.value ?? "";
  const done = world.has(todoEntity, Done);

  // materialize() creates entities from JSX and parents them under todoEntity
  materialize(world, [
    // Toggle checkbox — Clickable makes it emit Clicked on click,
    // ToggleButton tags it so ToggleTodoSystem can find it
    <Entity>
      <DOMElement tag="button" />
      <TextContent value={done ? "\u2611" : "\u2610"} />
      <Style border="none" background="none" fontSize="18px"
             cursor="pointer" padding="0" />
      <Clickable />
      <ToggleButton />
    </Entity>,

    // Label showing the todo text
    <Entity>
      <DOMElement tag="span" />
      <TextContent value={text} />
      <Style flex="1"
             textDecoration={done ? "line-through" : "none"}
             opacity={done ? "0.5" : "1"} />
    </Entity>,

    // Delete button
    <Entity>
      <DOMElement tag="button" />
      <TextContent value={"\u00d7"} />
      <Style border="none" background="none" color="#ef4444"
             fontSize="18px" cursor="pointer" padding="0 4px" />
      <Clickable />
      <DeleteButton />
    </Entity>,
  ], todoEntity);
}

const TodoEnterSystem = defineReactiveSystem({
  name: "TodoEnterSystem",
  query: Entities.with([TodoText]),
  onEnter(world, entities) {
    for (const entity of entities) {
      // Turn the todo entity itself into a flex row
      world.add(entity, DOMElement({ tag: "div" }));
      world.add(entity, Style({
        display: "flex", alignItems: "center", gap: "8px",
        padding: "6px 0", borderBottom: "1px solid #e5e7eb",
      }));
      buildTodoRow(world, entity);
    }
  },
});
```

The `TodoEnterSystem` watches for new `TodoText` entities. When one appears, it adds a `DOMElement` (making it a visible div) and calls `buildTodoRow` to create its children. The flush loop takes care of the rest — `DOMElementSystem` creates DOM nodes, `ClickableSystem` wires click handlers.

## 4. Toggle and Delete via Parent Traversal

When a toggle or delete button is clicked, the system walks up to the parent entity (the todo) using `world.getParent()`:

```ts
// When a toggle button is clicked, walk up to the parent todo and flip Done.
const ToggleTodoSystem = defineReactiveSystem({
  name: "ToggleTodoSystem",
  query: Entities.with([Clicked, ToggleButton]),
  onEnter(world, entities) {
    for (const e of entities) {
      world.remove(e, Clicked);  // Consume the click event
      // The button is a child of the todo entity — walk up one level
      const todoEntity = world.getParent(e);
      if (todoEntity == null) continue;

      // Toggle is just adding or removing a marker component
      if (world.has(todoEntity, Done)) {
        world.remove(todoEntity, Done);
      } else {
        world.add(todoEntity, Done());
      }
    }
  },
});

// When a delete button is clicked, remove the parent todo entity entirely.
// removeEntity recursively removes children, and DOMElementSystem's onExit
// cleans up the DOM nodes automatically.
const DeleteTodoSystem = defineReactiveSystem({
  name: "DeleteTodoSystem",
  query: Entities.with([Clicked, DeleteButton]),
  onEnter(world, entities) {
    for (const e of entities) {
      world.remove(e, Clicked);
      const todoEntity = world.getParent(e);
      if (todoEntity == null) continue;
      world.removeEntity(todoEntity);
    }
  },
});
```

This is the key advantage of entity-per-item: toggling is just `world.add(entity, Done())` or `world.remove(entity, Done)`. Deleting is `world.removeEntity(entity)` — the framework recursively removes children and cleans up their DOM nodes automatically.

No arrays to copy, no indices to track, no full-list re-renders.

## 5. Update Visuals on State Change

When `Done` is added or removed, the todo's children need rebuilding to reflect the new style. This requires **two systems** — one for each direction:

```ts
// Helper: tear down a todo's children and rebuild them
function rebuildTodoRow(world, entity) {
  for (const child of world.getChildren(entity)) {
    world.removeEntity(child);
  }
  buildTodoRow(world, entity);
}

// Fires when Done is ADDED — the entity enters the [TodoText, Done] query
const TodoMarkedDoneSystem = defineReactiveSystem({
  name: "TodoMarkedDoneSystem",
  query: Entities.with([TodoText, Done]),
  onEnter(world, entities) {
    for (const entity of entities) {
      rebuildTodoRow(world, entity);
    }
  },
});

// Fires when Done is REMOVED — the entity enters the [TodoText, !Done] query
const TodoUnmarkedDoneSystem = defineReactiveSystem({
  name: "TodoUnmarkedDoneSystem",
  query: Entities.with([TodoText, DOMElement]).without([Done]),
  onEnter(world, entities) {
    for (const entity of entities) {
      rebuildTodoRow(world, entity);
    }
  },
});
```

Why two systems? Reactive queries fire `onEnter` when an entity starts matching and `onExit` when it stops. The `onUpdate` callback only fires when a **query component** changes — adding `Done` doesn't trigger `onUpdate` for a `[TodoText, DOMElement]` query because `Done` isn't part of that query. By including `Done` in the query directly, we get `onEnter` callbacks exactly when the completion state changes.

This only rebuilds children of the *changed* todo entity — not the entire list.

## 6. Handle Adding Todos

When the Add button is clicked, read the input and create a new entity as a child of the list container:

```ts
import { Clicked, getDOMElement } from "@ecs-test/dom";

const AddTodoSystem = defineReactiveSystem({
  name: "AddTodoSystem",
  query: Entities.with([Clicked, AddButton]),
  onEnter(world, entities) {
    for (const e of entities) world.remove(e, Clicked);

    // No built-in input component — use getDOMElement to access the raw DOM node
    const inputEntity = world.query(TodoInput)[0];
    if (inputEntity == null) return;
    const inputEl = getDOMElement(world, inputEntity) as HTMLInputElement;
    if (!inputEl) return;

    const text = inputEl.value.trim();
    if (!text) return;
    inputEl.value = "";

    // Create the todo as a child of the list container.
    // TodoEnterSystem will pick it up and build its DOM children.
    const listEntity = world.query(TodoListContainer)[0];
    if (listEntity == null) return;
    const todo = world.createEntity(listEntity);
    world.add(todo, TodoText({ value: text }));
  },
});
```

Creating a todo is two lines: `createEntity(listEntity)` to parent it under the list, and `world.add(todo, TodoText(...))` to give it data. The `TodoEnterSystem` from step 3 handles the rest.

## 7. Wire and Run

<div data-code-tabs>

```tsx
import { World } from "@ecs-test/ecs";
import { registerDOMSystems } from "@ecs-test/dom";

const world = new World({
  externals: {
    createElement: document.createElement.bind(document),
    rootContainer: document.getElementById("root")!,
    window,
    console,
  },
});

registerDOMSystems(world);
world.registerSystem(TodoEnterSystem);
world.registerSystem(ToggleTodoSystem);
world.registerSystem(TodoMarkedDoneSystem);
world.registerSystem(TodoUnmarkedDoneSystem);
world.registerSystem(DeleteTodoSystem);
world.registerSystem(AddTodoSystem);

// materialize(world, ui) — the shell from step 2

// Seed initial todos under the list container.
// Each is just an entity with TodoText, optionally Done.
const listEntity = world.query(TodoListContainer)[0];

const t1 = world.createEntity(listEntity);
world.add(t1, TodoText({ value: "Learn about ECS entities" }));
world.add(t1, Done());  // This one starts completed

const t2 = world.createEntity(listEntity);
world.add(t2, TodoText({ value: "Build a todo app" }));

world.flush();
```
